# Contributing to XRay Proxy Validator

Thank you for your interest in contributing to the XRay Proxy Validator project! This document provides guidelines and instructions for contributing.

## Getting Started

1. **Fork the repository** - Click the "Fork" button on GitHub
2. **Clone your fork** - `git clone https://github.com/YOUR_USERNAME/xray-proxy-validator.git`
3. **Create a branch** - `git checkout -b feature/your-feature-name`
4. **Make your changes** - Implement your feature or fix
5. **Commit your changes** - `git commit -am 'Add your feature'`
6. **Push to your fork** - `git push origin feature/your-feature-name`
7. **Submit a Pull Request** - Open a PR on the main repository

## Code Style

- Follow PEP 8 for Python code
- Use meaningful variable and function names
- Add docstrings to functions and classes
- Keep lines to a reasonable length (80-100 characters)

## Testing

- Test your changes thoroughly
- Test with multiple proxy types (VLESS, VMess, Trojan, ShadowSocks)
- Ensure no regressions are introduced

## Commit Messages

- Use clear and descriptive commit messages
- Start with a verb (Add, Fix, Update, Remove, Refactor)
- Example: `Fix IP reputation check for DNSBL queries`

## Pull Request Guidelines

1. **Describe your changes** - Explain what your PR does and why
2. **Reference issues** - Link related issues using "Fixes #123"
3. **Keep it focused** - One feature per PR is ideal
4. **Test before submitting** - Ensure all tests pass
5. **Respond to reviews** - Be open to feedback and suggestions

## Report Issues

Found a bug? Please report it:

- Go to the [Issues](https://github.com/kort0881/xray-proxy-validator/issues) page
- Click "New issue"
- Provide a clear title and description
- Include steps to reproduce if possible
